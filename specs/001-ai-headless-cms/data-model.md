# Data Model

## Bounded Contexts
1. **CMS Context (Payload CMS)**: Manages tenants, users, dynamic schemas, and content items.
2. **AI Agent Context (AI Microservice)**: Manages AI conversation state, LLM interactions, and generative workflows.

## Collections (CMS Context - Payload CMS)

### ContentType
- Stores JSON schemas for content dynamically generated by AI or users.

### ContentItem
- `id`: String/UUID
- `tenant`: Relationship to `Tenant`
- `contentType`: Relationship to `ContentType`
- `title`: Text
- `content`: Blocks / JSON
- `status`: Select (draft, published)

### Tenants
- `id`: String/UUID
- `name`: Text
- `slug`: Text (unique)

### Users & APIKeys
- Standard Payload Auth collections, related to `Tenants`.

## Entities (AI Agent Context - AI Microservice)

### AIAgentSession (Aggregate Root)
- `id`: UUID
- `userId`: UUID (Reference to Payload User)
- `tenantId`: UUID (Reference to Payload Tenant)
- `context`: JSON (Conversation history)
- `status`: Enum (Active, Completed, Failed)
- `createdAt`: DateTime
- `updatedAt`: DateTime

## Relationships
- **Cross-Service**: The AI Microservice references `userId` and `tenantId` from the CMS but does not hold a hard foreign key constraint. It relies on the CMS providing valid tokens/IDs.
- **Enforcement**: Payload CMS handles read/write access control. The AI Microservice acts as an internal trusted client when writing generated content back to Payload.
